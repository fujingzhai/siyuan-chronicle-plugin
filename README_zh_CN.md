# 岁时记（Chronicle）

简体中文 | [English](./README.md)

岁时记是一款以时间为中心的思源笔记记录面板。它把同一年展开为年、季、月、周四种彼此协调的尺度，让经历自然地落在时间里，而不必被改造成任务、日程或数据库条目。

名字取《荆楚岁时记》之意。插件关注的不是“接下来必须做什么”，而是“这一段时间发生过什么”：时间轴始终是主体，类目只是可选的整理维度，需要展开叙述时再连接思源原生笔记。

## 设计宗旨

- **时间优先**：以年、季、月、周组织回忆，不另设类目视图，不让分类体系反客为主。
- **安静克制**：尽量淡化表格和工具栏，只保留必要的信息、颜色与操作反馈。
- **记录而非督促**：活动没有完成状态，不把生活记录变成待办清单。
- **原生笔记**：需要长文时连接真实的思源文档，而不是把正文困在插件私有数据库里。
- **数据边界清楚**：活动与笔记互相独立；删除、移动或改名都有明确而保守的处理方式。

## 主要功能

- 年、季、月固定一屏，周独立纵向滚动；默认列宽比例为 `1 : 1 : 3 : 5`，可拖动分隔线自定义并自动记忆。
- 活动可记录标题、类目、时间、可选的具体日期、备注，以及一篇或多篇绑定笔记。
- 年和季默认一项一行；月和周按标题长度紧凑排列。
- 同一时间格内可手动拖动排序；也可把活动拖到另一时间格，活动时间与具体日期会随落点自动调整。
- 点击任意年、季、月、周标题，可创建或打开对应的时间笔记；已有时间笔记以文档图标表示。
- 设置中可拖动调整类目顺序，并用系统取色器选择颜色。
- 面板内快捷键：`T` 定位今天，`N` 新建活动，`S` 打开设置，`←` / `→` 切换年份。输入文字或打开编辑对话框时不会误触，快捷键也不会注册为思源全局快捷键。

## 笔记绑定规则

活动和笔记是两层数据。活动可以完全不绑定笔记，也可以连接已有文档或由岁时记新建文档。

- **移动文档**：绑定以思源文档块 ID 为准，文档移动到其他路径或笔记本后仍能正常打开。
- **重命名文档**：岁时记会根据文档 ID 定期核对，自动刷新活动中显示的绑定笔记标题。
- **删除绑定文档**：只解除失效绑定，活动本身不会删除。
- **删除时间文档**：对应的文档图标会消失，该时间恢复为可点击新建的状态。
- **删除活动**：未绑定笔记时直接删除；存在绑定笔记时可选择是否同时删除这些笔记，默认不勾选。

## 默认笔记本与迁移

首次使用时，插件会优先使用名为“岁时记”的笔记本；不存在则自动创建。你也可以在设置中改用其他笔记本。

切换默认笔记本时，岁时记只迁移两类带有插件标记的文档：时间笔记树，以及通过“新建并绑定”创建的活动笔记。用户从其他位置绑定的既有文档不会被移动。若目标笔记本存在同路径文档，迁移会在动手前中止并提示，避免覆盖或只迁移一半。

## 数据安全

活动数据以 JSON 存放在思源工作空间的插件数据目录中。写入采用串行队列，连续拖动或快速编辑不会让较早的异步写入覆盖较新的状态；每次写入前还会保留上一份有效数据到 `chronicle.backup.json`。笔记本中的文档仍是完整的思源原生文档，正常参与同步、搜索、历史与快照。

## 构建与本地安装

```bash
pnpm install
pnpm build          # 构建 dist/ 并生成 package.zip
pnpm make-install   # 安装到 ~/SiYuan/data/plugins/siyuan-chronicle-plugin
```

项目与插件包标识为 `siyuan-chronicle-plugin`；思源集市中文名为“岁时记”，英文名为“Chronicle”。
